# Володичева Варвара БПИ-247
### проект написан на C#

## 1. Описание архитектуры системы
Система реализована на основе **микросервисной архитектуры** с использованием подхода **N-tier Architecture**.

Архитектура состоит из трёх независимых микросервисов:
- **ApiGateway** — единая точка входа в систему. Принимает запросы от пользователей и управляет взаимодействием между сервисами.
- **FileStorageService** — отвечает за приём и хранение файлов, а также вычисление хеш-значений загруженных работ.
- **FileAnalysisService** — выполняет анализ файлов и определяет наличие плагиата на основе хешей.

Микросервисы взаимодействуют между собой по протоколу **HTTP** в синхронном режиме, есть централизованное управление через API Gateway.

### 2. Пользовательский сценарий: загрузка работы
1. Пользователь отправляет файл на проверку через **ApiGateway**.
2. ApiGateway принимает запрос и передаёт файл в **FileStorageService**.
3. FileStorageService:
   - сохраняет файл,
   - вычисляет хеш содержимого,
   - возвращает идентификатор и хеш файла.
4. ApiGateway передаёт полученные данные в **FileAnalysisService**.
5. FileAnalysisService проверяет, встречался ли данный файл ранее, и формирует результат анализа.
6. ApiGateway возвращает пользователю результат проверки.

### Пользовательский сценарий: получение отчётов по заданию
1. Пользователь отправляет запрос на получение отчётов по заданию через **ApiGateway**.
2. ApiGateway запрашивает данные у **FileAnalysisService**.
3. FileAnalysisService возвращает список работ и результаты анализа.
4. ApiGateway передаёт полученную информацию пользователю.

### Технические сценарии взаимодействия сервисов
- **ApiGateway → FileStorageService**  
  Используется для загрузки файлов и получения их хеш-значений.
- **ApiGateway → FileAnalysisService**  
  Используется для анализа файлов и формирования отчётов.

## О запуске 
Если значение поля `isPlagiat` равно false, работа считается оригинальной.
Если значение поля `isPlagiat` равно true, работа определяется как плагиат,
так как её содержимое совпадает с ранее загруженной работой.

Для работы программы должны быть установлены следующие пакеты: 
 - Docker.DotNet
 - Microsoft.Graph.Core
 - Swashbuckle.AspNetCore.Swagger
 - Swashbuckle.AspNetCore.SwaggerGen
 - Swashbuckle.AspNetCore.SwaggerUI
